### Notes

Built a static contact form (no JS yet). Used semantic HTML throughout: form element, labels paired with inputs via for/id matching, the required attribute on every input for built-in validation, type='email' on the email field for browser-level email format checking, textarea for the longer message field, and a submit button. Then styled the form as a dark card with flexbox vertical layout, custom inputs and textarea styling, and an amber submit button with a hover transition.

### Code

```html
<!-- contact.html -->
<form id="contact-form">
    <label for="name-input">Name</label>
    <input id="name-input" type="text" required>

    <label for="email-input">Email</label>
    <input id="email-input" type="email" required>

    <label for="message-input">Message</label>
    <textarea id="message-input" required></textarea>

    <button type="submit">Send</button>
</form>
```

```css
/* style.css */
form {
    display: flex;
    flex-direction: column;
    gap: 12px;
    max-width: 480px;
    background-color: #2d2d44;
    padding: 24px;
    border-radius: 8px;
}
input, textarea {
    padding: 10px;
    font-size: 16px;
    border: 1px solid #1a1a2e;
    border-radius: 4px;
    background-color: #e0e0e0;
    color: #1a1a2e;
}

    padding: 12px 16px;
    font-size: 16px;
    font-weight: bold;

    border-radius: 4px;
    transition: background-color 0.2s ease;
}
button:hover {
    background-color: #d97706;
}
```

### What I learned

HTML form elements give you accessibility for free if you pair labels with inputs correctly via for/id. The required attribute and type='email' provide client-side validation with zero JavaScript. Flexbox with flex-direction: column and gap is the cleanest way to lay out a vertical form.
