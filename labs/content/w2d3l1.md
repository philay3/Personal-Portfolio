### Notes

Warmup activity. Took the Greet Me page from Day 2 and broke it four different ways on purpose, documenting each as symptom, cause, fix. The goal: build pattern recognition for the most common DOM and JS failure modes.

**Break 1: Typo in `getElementById("nam-input")`**

Symptom: Console error `Uncaught TypeError: Cannot read properties of null (reading 'value')`, pointing to the line where `nameInput.value` is read.

Cause: `getElementById("nam-input")` doesn't find a matching element, so it returns `null`. Reading `.value` on `null` throws.

Fix: Restore the correct id `"name-input"`. The id string in JS must match the id attribute in HTML exactly.

**Break 2: Commented out `event.preventDefault()`**

Symptom: The greeting flashes for a split second, then disappears. The URL gains a `?name=...` query string. Page appears to reload.

Cause: A form's default submit behavior is to send the data to the server (or, with no action attribute, reload the current page). The handler runs, writes the greeting, then the page reloads and wipes the DOM.

Fix: Restore `event.preventDefault();` inside the submit handler so the browser doesn't run its default form submission.

**Break 3: `nameInput.values` (plural)**

Symptom: No error. The greeting reads `Hello, undefined!`.

Cause: `.values` is not a property on an input element, so accessing it returns `undefined`. String concatenation with `undefined` coerces it to the string `"undefined"`.

Fix: Restore `nameInput.value`. This is the silent failure mode: no red error, just wrong output. Catch these with `console.log(nameInput.value)` before using it.

**Break 4: `<script>` moved into `<head>`**

Symptom: Console error about `addEventListener` or `getElementById` returning null. Submit button does nothing.

Cause: Scripts in `<head>` run before the body has been parsed, so any element the script tries to grab doesn't exist yet. `getElementById` returns `null`, then `.addEventListener` on null throws.

Fix: Move the script back to the bottom of `<body>`, OR add `defer` to the script tag in head: `<script src="greet.js" defer></script>`. The `defer` attribute tells the browser to wait until HTML parsing finishes before running the script.

### What I learned

Most bugs fit one of three patterns: name doesn't exist (typo or scope issue), value is the wrong type (null, undefined, wrong method name), or load order is wrong (scripts running before the DOM exists). Breaking code on purpose with a known intent makes the symptom-cause-fix arc obvious. Reading that arc enough times means you start recognizing it instantly when bugs happen for real.
