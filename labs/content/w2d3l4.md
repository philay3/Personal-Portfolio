### Notes

Five small explorations of Chrome DevTools to learn what each tab does. No code to write, just hands-on inspection of a live page.

**Challenge 1: Console tab**

- `2 + 2` returned `4`
- `document.title` returned the page's title string (whatever's in the `<title>` tag)
- `document.title = "I changed the title!"` updated the browser tab text immediately

The Console isn't just for reading errors. You can execute JavaScript against the live page.

**Challenge 2: Elements tab**

Changing a color value in the Styles panel updated the page instantly. No refresh. Adding `text-decoration: underline;` worked the same way. DevTools applies CSS live.

**Challenge 3: Box Model**

Inspecting an element in the Computed tab showed a nested-box diagram: content size in the middle, surrounded by padding, surrounded by border, surrounded by margin. Each side has its own value. This is the actual rendered box, not what's declared in CSS, so you can see what the browser ended up computing.

**Challenge 4: Network tab**

After refresh, the Network tab listed every resource the page requested: HTML, CSS, JS, images, fonts, favicon. Each row shows file size, load time, and HTTP status. Clicking a CSS file opens its source. Useful for catching 404s on broken paths and seeing how heavy a page actually is.

**Challenge 5: Live editing**

Edited a paragraph's text in the Elements panel and the page updated. Refreshed and the edits vanished.

Why: DevTools edits live in the browser's in-memory copy of the DOM. The source files on disk (or on the server) are untouched. Refreshing re-fetches and re-parses the original files, overwriting your in-memory changes. DevTools is for experimenting and inspecting, not editing source.

### What I learned

DevTools is five tools in one. Console is a live JS REPL. Elements is a live DOM and CSS inspector. Network is a request log. Computed shows what the browser actually rendered. Every edit is in-memory and disappears on refresh. The mental model: DevTools is a flight simulator, not a flight. You save the lesson by carrying the fix back to your source files.
