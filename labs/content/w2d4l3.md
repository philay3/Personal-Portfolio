### Notes

Built `books.html` and `books.js` with the four starter books, then iterated on the rendering through three steps.

**Step 1: Adding more books**

After adding two more book objects to the array and refreshing, both new books appeared automatically. The rendering loop doesn't care how many items are in the array. This is the whole point of data-driven rendering: the data shape stays the same, the code stays the same, the output scales with the data.

**Step 2: Title in `<strong>`, author in normal text**

Two approaches considered.

Verbose approach using `createElement`:

```javascript
for (const book of books) {
  const li = document.createElement("li");
  const titleEl = document.createElement("strong");
  titleEl.textContent = book.title;
  li.appendChild(titleEl);
  li.appendChild(document.createTextNode(` -- ${book.author}`));
  list.appendChild(li);
}
```

Concise approach using `innerHTML`:

```javascript
for (const book of books) {
  const li = document.createElement("li");
  li.innerHTML = `<strong>${book.title}</strong> -- ${book.author}`;
  list.appendChild(li);
}
```

Picked the concise approach because the book data is hardcoded in my own JS file. No user input, no XSS risk. The moment user-supplied input is interpolated (like task titles in the todo app), `innerHTML` becomes dangerous and `createElement` + `textContent` is the safe path.

**Step 3: Wrapping in `renderBooks(bookArray)`**

```javascript
function renderBooks(bookArray) {
  list.innerHTML = "";  // clear existing items first
  for (const book of bookArray) {
    const li = document.createElement("li");
    li.innerHTML = `<strong>${book.title}</strong> -- ${book.author}`;
    list.appendChild(li);
  }
}

renderBooks(books);

const moreBooks = [
  { title: "Bird by Bird", author: "Anne Lamott" },
  { title: "The Pragmatic Programmer", author: "Andy Hunt & Dave Thomas" }
];
renderBooks(moreBooks);
```

Calling `renderBooks` twice replaces the list each time (because of the `list.innerHTML = ""` at the top). Only the second array's books are visible after both calls run. That clear-and-redraw pattern is exactly how the todo app in Exercise 5 works, and exactly how React re-renders on state change.

### What I learned

Before functions, rendering logic and data were tangled together. After functions, the data is the input and the rendering is the function. Same function, different data, different output. That separation, plus the clear-and-redraw pattern, is the entire mental model behind data-driven UIs.
