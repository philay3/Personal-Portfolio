### AI vs me: the running list

Throughout the day I kept a running record of every change I made to AI's output. Each item is a place AI's first draft got modified, with the reason behind the change:

- Generic intro paragraph ("Welcome to my portfolio. I am a developer who..."). Rewrote in my own voice referencing Next Chapter and the bootcamp's premise. The whole point of the portfolio is voice. Generic intros are forgettable.
- Placeholder project cards with lorem ipsum. Replaced with one real card for Digital Footprinter (real screenshot, real description, real links). Hiring managers can tell the difference between a real project and a fake one.
- Class names like `.section-1` and `.section-2`. Renamed to `.biography`, `.projects`, `.articles`, `.gallery`. Class names should read like English. Future-me debugging at 11pm shouldn't have to count sections.
- Hardcoded social URLs as `#`. Replaced with my real GitHub and LinkedIn. The site is live; placeholders embarrass.
- `transition: all 0.3s ease` on hover states. Replaced `all` with specific properties (`background-color`, `color`, `border-color`). `transition: all` animates unrelated property changes when you don't want them to.
- `clamp()` for responsive font sizing on every heading. Kept it on the H1 and the article card thumb only. Switched everything else to plain `rem` values I could reason about.
- Multiple media queries (one per section). Consolidated to one breakpoint at 700px for the project card stack. Most of the layout is already fluid via flexbox and max-width, so the extra breakpoints were noise.
- `cursor: pointer` on the body. Deleted. Made the whole page feel clickable. Active anti-pattern.
- Smooth scroll implemented in JavaScript with custom easing. Replaced with `html { scroll-behavior: smooth; }` in CSS. One line handles it natively.
- Scroll spy using a scroll event listener with `getBoundingClientRect` math. Rewrote using the Intersection Observer API. Scroll events fire constantly and require throttling. Intersection Observer is the modern, performant pattern.
- Initial scroll spy highlighted multiple links at once. Tightened `rootMargin` to `-40% 0px -40% 0px`. Without it, two sections were considered "intersecting" whenever the viewport was tall enough to show both.
- Contact form section in the original scaffold. Deleted entirely. Out of scope per the Day 1 scope doc. Footer social links cover the contact case without requiring a backend.

### What I learned

A change-tracking log is the artifact, not a chore. It looks like busywork while you're keeping it, and like the most important document on the project the moment someone asks "what did you actually decide?" Without it, the Day 4 presentation would have been "I built a portfolio." With it, it's "I built a portfolio and here are twelve specific choices I made along the way."

The other shape of the lesson: the value of AI isn't the first draft. The first draft is roughly as good as a senior dev's first stab. The value is in the editing pass, where my judgment as a builder gets to assert itself in concrete, defensible ways.
