### Notes

Day 2 was about getting the site functional. Day 3 was about making it sharp. The polish exercise was a 15-item checklist walked through in order: content first, then visual, then functionality. I fixed each item as I found it.

### Content (items 1-5)

- Name and title consistent across the page. "Phil Anthony" and "Software Engineer" everywhere they appear, no abbreviations or typos.
- Bio is in my voice, not AI's. The Next Chapter intro reads like me talking, not a template. Reread it three times and tightened wording.
- Project description is concrete, not generic. Digital Footprinter card names specific technical details: canvas hash, WebRTC IPs, fonts, user agent, screen size, timezone. No "I built a cool tool" filler.
- Article titles and dates are correct. Cross-checked against the actual Substack posts.
- No "Lorem ipsum" or "TODO" left anywhere. Did a global search across `index.html` for "lorem", "TODO", "placeholder". Came up empty.

### Visual (items 6-10)

- Section spacing is consistent. Initial CSS had hardcoded pixel values for padding scattered across the section rules because I'd edited piecemeal during Day 2. Consolidated everything into CSS custom properties defined at the top of the file (`--space-xs` through `--space-xl`) so any future spacing tweak happens in one place. Biography gets `--space-xl` since it's the hero; other sections get `--space-lg` to feel like a step down from the landing zone.
- Colors are intentional and used consistently. The emphasis color (brick red) appears in exactly the places it should: links and the active scroll-spy state. The accent green is reserved for button and card hover states. Nowhere else.
- Type hierarchy is clear. H1 (name in hero) > H2 (section headings) > H3 (project name) > body. Sizes follow the hierarchy, not the order I wrote them in.
- Images load reliably and at the right size. Portrait sized appropriately for the hero. Digital Footprinter screenshot crisp at desktop and mobile.
- Hover and active states feel right. Nav links shift to the emphasis color cleanly, project card buttons invert on hover, no jarring color jumps.

### Functionality (items 11-15)

- Scroll spy works on every section. Tested by scrolling slowly and watching the active link change. Tested by clicking each nav link and confirming the right one stays highlighted.
- Smooth scroll works on anchor clicks. Click "Projects" in the nav and the page glides instead of jumping. Single CSS property: `html { scroll-behavior: smooth; }`.
- All external links open correctly. GitHub, LinkedIn, Substack articles, Digital Footprinter live site, Digital Footprinter source. Clicked every one.
- No console errors on page load. Opened DevTools Console on a hard reload. Zero red errors, zero warnings I cared about.
- Page loads fast. Used the Network tab in DevTools to confirm: under 1 second to interactive on Wi-Fi. No oversized images, no blocking scripts.

### Sample AI prompt I used during the polish pass

```
The spacing values in my CSS are scattered as raw pixel values across
multiple section rules. I want to consolidate them into CSS custom
properties so I have one place to change them. Here's the relevant CSS:
[pasted my .biography, .projects, .articles, .gallery blocks]

Suggest a clean way to define a spacing scale and apply it.
```

AI's answer was to define a set of `--space-*` variables in `:root` and reference them from each section's padding. I implemented it as a five-step scale (`xs` through `xl`) so I had room to draw from, not just the two values I needed today. Future-me will use the rest.

### What I learned

A polish pass is a different kind of work from a build pass. Build is generative: typing new code, making new choices. Polish is mostly subtractive: catching inconsistencies, removing roughness, normalizing. Both are necessary. The temptation is to skip polish because the site "already works" after the build. The reality is that polish is where the site stops feeling like a draft.

The 15-item checklist worked because it gave me permission to stop. Without it, polish is infinite. With it, I had a defined finish line.
