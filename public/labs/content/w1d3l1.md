### Notes

Given a chain of if/else if/else statements checking a weather string, predicted what each input would log. With weather of 'rainy' the second branch hits and prints 'Grab an umbrella!' With 'cloudy' it falls all the way to the else and prints 'Check the forecast.' With 'Rainy' (capital R) it ALSO falls to else because `===` is case sensitive.

### Code

```javascript
const weather = "rainy";
if (weather === "sunny") console.log("Wear sunglasses!");
else if (weather === "rainy") console.log("Grab an umbrella!");
else console.log("Check the forecast.");
```

### What I learned

`===` checks for an EXACT match, including type and case. Same idea as the comparison operators in PowerShell that have bit me before. Strict equality has no surprises but no leniency either.
