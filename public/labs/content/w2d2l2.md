### Notes

First DOM manipulation lab. Wrote greet.html with a form and an empty paragraph for output, plus greet.js to wire them up. In the JS: grabbed all three elements (form, input, paragraph) with getElementById, added a submit event listener on the form, called event.preventDefault() to stop the page reload, read the input value, and wrote it into the paragraph via textContent.

Then ran three investigations to understand what each piece was doing. (1) Commented out event.preventDefault(). Page tried to submit to a server, refreshed, wiped JS state, greeting flashed and disappeared. That confirmed why preventDefault is mandatory. (2) Switched textContent to innerHTML with a `<strong>` tag in the message. Name rendered bold because innerHTML actually parses HTML. (3) Typed `<em>injected</em>` as the name with innerHTML still in place. Page rendered it as italics. If that had been a `<script>` tag, the browser would have executed it. That's exactly how XSS vulnerabilities are born. Switched back to textContent and the same input rendered as literal angle brackets, neutralizing the threat.

### Code

```html
<!-- greet.html -->
<form id="greet-form">
    <label for="name-input">What's your name?</label>
    <input id="name-input" type="text" required>
    <button type="submit">Greet me</button>
</form>

<p id="greeting"></p>

<script src="greet.js"></script>
```

```javascript
// greet.js
const form = document.getElementById("greet-form");
const nameInput = document.getElementById("name-input");
const greeting = document.getElementById("greeting");

form.addEventListener("submit", (event) => {
    event.preventDefault();
    const name = nameInput.value;
    greeting.textContent = `Hello, ${name}!`;
});
```

### What I learned

event.preventDefault() is mandatory in any client-side form handler, otherwise the browser reloads and your JS state vanishes. textContent renders user input as literal text (safe). innerHTML parses input as HTML (powerful, but the entry point for XSS attacks). Default to textContent unless you specifically need HTML in your output. Critical concept for any environment that processes user input, especially regulated ones like healthcare.
