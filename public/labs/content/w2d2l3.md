### Notes

Reused the Greet Me pattern on a slightly bigger form. Two inputs (favorite color as text, years coding as number with min='0'), one summary div for output. The JS followed the exact same five-step pattern: grab elements, listen for submit, prevent default, read values, write the summary string. Used template literals (backticks with `${}` interpolation) to inject both variables into one clean string.

### Code

```html
<!-- survey.html -->
<form id="survey-form">
    <label for="fav-color">Favorite color?</label>
    <input id="fav-color" type="text" required>

    <label for="years-coding">Years you've been coding?</label>
    <input id="years-coding" type="number" min="0" required>

    <button type="submit">See my summary</button>
</form>

<div id="summary"></div>
```

```javascript
// survey.js
const form = document.getElementById("survey-form");
const colorInput = document.getElementById("fav-color");
const yearsInput = document.getElementById("years-coding");
const summary = document.getElementById("summary");

form.addEventListener("submit", (event) => {
    event.preventDefault();

    const color = colorInput.value;
    const years = yearsInput.value;

    summary.textContent = `Your favorite color is ${color} and you've been coding for ${years} year(s).`;
});
```

### What I learned

Once you have the grab-listen-prevent-read-write pattern, you can apply it to any form regardless of size. Template literals are dramatically cleaner than string concatenation when injecting multiple variables. Backticks plus `${}` replaces a wall of plus signs and quotes.
