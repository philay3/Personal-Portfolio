### Notes

Picked Option A (a counter with +1, -1, and Reset buttons) but styled the whole thing as a calculator module to lock in the layout vocabulary. Red background page, black body with rounded corners and a shadow, teal digital display for the count, three buttons in a horizontal flex row. JS uses a count variable for state, three event listeners that each mutate the state and update the display via textContent.

```javascript
// counter.js
let count = 0;

const display = document.getElementById("display");
const btnMinus = document.getElementById("btn-minus");
const btnReset = document.getElementById("btn-reset");
const btnPlus = document.getElementById("btn-plus");

btnPlus.addEventListener("click", () => {
    count++;
    display.textContent = count;
});

btnMinus.addEventListener("click", () => {
    count--;
    display.textContent = count;
});

btnReset.addEventListener("click", () => {
    count = 0;
    display.textContent = count;
});
```

### What I learned

State lives in a variable. Events trigger functions. Functions update the state and re-render the relevant DOM. That trio is the core of every interactive page, framework or no framework. Same pattern, different syntax depending on the tool. The calculator styling is just dressing on the same machinery.
