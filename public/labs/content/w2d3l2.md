### Notes

Got five JavaScript snippets, each with a different error. Read the error message in each, identified the type (ReferenceError, TypeError, or SyntaxError), and worked out the fix.

**Error A: ReferenceError on line 3**

`username` was never declared. The variable is `userName` (capital N). JavaScript is case-sensitive.

Fix: Change `username` to `userName`.

**Error B: TypeError on line 2**

`fruits` is `undefined` (declared with no value), and you can't read `.length` on undefined.

Fix: Assign a value, e.g. `let fruits = [];` or `let fruits = ["apple", "banana"];`.

**Error C: SyntaxError on line 4**

There's an extra closing brace `}` with no matching opening brace. The function already closed on line 3.

Fix: Delete the extra `}` on line 4.

**Error D: TypeError on line 1**

`getElementByID` isn't a real method. The correct name is `getElementById` (lowercase `d`).

Fix: Change `getElementByID` to `getElementById`.

**Error E: SyntaxError on line 1**

The string opens with `"` but never closes. The line ends without a matching quote before the semicolon.

Fix: Add the closing quote: `const message = "Hello, World!";`.

### What I learned

Three error types, three signatures:

- **ReferenceError**: the name doesn't exist (typo, scope, or never declared)
- **TypeError**: the value exists but you're using it wrong (null, undefined, wrong method name)
- **SyntaxError**: JavaScript can't even parse the code (missing or extra brackets, quotes, commas)

Recognizing the error type from the message alone narrows the search dramatically before you even look at the code.
