### Notes

Two programs that ran without errors but produced wrong output. Both required adding `console.log` between steps to find where the actual value diverged from the expected value.

**Problem 1: Average calculation**

Expected average for 80, 90, 70: 80. Actual output: 120. Adding `console.log` after each variable assignment revealed:

- score1: 80 (correct)
- score2: 90 (correct)
- score3: 70 (correct)
- sum: 240 (correct)
- average: 120 (wrong, expected 80)

The bug was on the average line. Dividing `sum` (240) by 2 gives 120. Dividing by 3 (because there are 3 scores) gives 80.

### Code

```javascript
const average = sum / 3;
```

After fix, output reads `Your average score is: 80`. Matches expected.

**Problem 2: Discount calculation**

`applyDiscount(150)` returned 15 instead of 135. The bug: `price * 0.10` calculates the discount amount (the 10% being taken off), not the discounted price. A 10% discount means the customer pays 90% of the price.

Fix:

```javascript
const discounted = price * 0.90;
```

Or, more readable:

```javascript
const discounted = price - (price * 0.10);
```

After fix: `applyDiscount(150)` returns 135. `applyDiscount(80)` returns 72.

### What I learned

The code ran without errors. This is a logic bug, not a syntax bug. The only way to catch it is by knowing what the right answer should be, then verifying the code produces it. Don't guess where the bug is. Instrument with `console.log` between steps and the divergence point shows itself.
