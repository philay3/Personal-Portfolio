### Notes

Four short AI-generated programs, each with a bug. Read the code, predicted the bug, ran it, and fixed it.

**Bug 1: Greeting Generator**

The id is misspelled in `document.getElementById("greetng")`. The HTML has `id="greeting"`.

Fix:

```javascript
document.getElementById("greeting").textContent = greeting;
```

Console showed `Cannot read properties of null (reading 'textContent')` because `getElementById` returned null.

**Bug 2: Color Changer**

`style.colour` uses British spelling. The CSS and DOM property is `color` (American). Setting an unknown property on `style` does nothing and throws no error.

Fix:

```javascript
document.querySelector("h1").style.color = selectedColor;
```

Why no error: `style` is a plain object. Setting `style.colour = "red"` just creates a meaningless property. The browser only reads `style.color`, so nothing visible happens. Silent failure category.

**Bug 3: Counter**

`textContent = "count"` assigns the string `"count"` (in quotes), not the variable `count`. Removing the quotes uses the variable's value.

Fix:

```javascript
document.getElementById("count").textContent = count;
```

Quotes turn anything into a literal string. Without quotes, JavaScript looks up the identifier.

**Bug 4: Temperature Message**

`prompt()` always returns a string. `"9" > "85"` is `true` because string comparison goes character by character: `"9"` comes after `"8"`. The input needed to be a number before being compared.

Fix:

```javascript
const temp = Number(prompt("Enter the temperature in Fahrenheit:"));
```

Or use `parseInt(prompt(...), 10)` if you want to allow input like `"75 degrees"` (parseInt grabs the leading number).

Defensive add-on: check `isNaN(temp)` and warn the user if they typed something that isn't a number.

### What I learned

AI writes the same kinds of bugs humans write: typos in IDs, regional spelling, quotes where they shouldn't be, type confusion across `prompt()` returns. Two of the four bugs (color and counter) failed silently with no console error. Those are the ones you only catch by testing against expected output, never by running once and assuming green.
