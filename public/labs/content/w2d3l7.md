### Notes

Three-file project (HTML, CSS, JS) for a simple to-do list with two bugs hidden across the files. Goal: find both, fix both, explain how I found them.

**Bug 1: HTML mismatched closing tag**

```html
<ul id="task-list">
</ol>   <!-- this should be </ul> -->
```

Symptom: Browsers are forgiving and often render the page anyway, but the DOM tree gets weird. Layout breaks subtly. The Elements panel in DevTools usually flags it.

Fix: Change `</ol>` to `</ul>`.

**Bug 2: CSS missing hyphen**

```css
alignitems: center;   /* not a valid property */
```

Symptom: Delete buttons don't align vertically with the task text inside each `<li>`. No error, because invalid CSS properties are just ignored.

Fix: `align-items: center;`

**The JS file was clean**

Read top to bottom, tracing what each line does. Verified the closure in `deleteBtn.onclick` captures the right `li` (each call to `addTask` creates a new `li` in its own scope). Verified `removeChild` is called on the parent (`taskList`) with the child (`li`) as the argument. No typos in DOM API calls. After fixing the HTML and CSS bugs, the page worked end to end, which confirmed the JS was good.

**How I found each bug**

HTML mismatch: Looking at the Elements panel after load, the DOM tree didn't match the source. That's a tag-mismatch tell. Going back to the source, `</ol>` jumped out.

CSS missing hyphen: Visually, the delete button wasn't centered with the task text. Inspected the `li` in DevTools, checked Computed styles, noticed `align-items` wasn't applied. Checked the source and saw `alignitems`.

### What I learned

Each language has its own failure signature. HTML breaks the DOM tree (visible in Elements). CSS silently ignores invalid properties (visible in Computed). JS throws or silently produces wrong output (visible in Console). When debugging a multi-file project, start with where the symptom is and walk back to the file responsible. Layout off goes to CSS. DOM weird goes to HTML. Logic wrong or error in console goes to JS.
