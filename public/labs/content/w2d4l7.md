### Notes

Attempted the localStorage persistence stretch goal on the todo app. The goal: tasks survive a page refresh.

Two-part change.

**Part 1: Save on every mutation**

Added one line after each place that modifies the `tasks` array (add, clear, checkbox toggle):

```javascript
localStorage.setItem("tasks", JSON.stringify(tasks));
```

**Part 2: Restore on page load**

Added one block at the top of the script, after the DOM references:

```javascript
const saved = JSON.parse(localStorage.getItem("tasks") || "[]");
for (const task of saved) {
  tasks.push(task);
}
renderTasks();
```

Tasks now survive a refresh. The `|| "[]"` fallback handles the first-ever page load when nothing has been saved yet, so `JSON.parse` always gets valid JSON.

**What I skipped**

Did not attempt the `.map` / `.filter` rewrite of the renderer. Want to be solid on for...of before adding new syntax on top.

### What I learned

`JSON.stringify` turns an array of objects into a string the browser can store. `JSON.parse` turns it back into a real array of objects when the page loads. Together with localStorage, that's a 2-line persistence layer. No backend, no database, no auth, just a place in the browser to put data that survives refresh. Good enough for any single-device app. Once a second device or user enters the picture, you need a real backend.
