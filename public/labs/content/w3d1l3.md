### Notes

Exercise 3 was the first real "ask AI to build me something" step in the project. The prompt had to be specific enough that the output was usable, but loose enough that I could keep my own voice in the result.

### Prompt I used

```
Build the HTML structure for a personal portfolio. Sections:

- Sticky top nav with links to: Biography, Projects, Articles, Gallery
- Hero / Biography section with a portrait image, an H1 with my name,
  a subtitle ("Software Engineer"), a 2-sentence intro paragraph,
  and a "See My Work" button that anchors to #projects
- Projects section as a list of cards. Each card has an image, an H3
  with the project name, a description paragraph, a "Stack:" line,
  and two links: Live Site and Source on GitHub
- Articles section: a heading and a list of article titles with dates,
  each one a link out to Substack
- Gallery section: heading only for now, with "Coming soon." as body
- Footer with a GitHub link and a LinkedIn link, plus a copyright line

Use semantic HTML (header, nav, main, section, footer, article).
Give each section an id so the nav can anchor to it.
Class names should be descriptive and lowercase-hyphenated.
No CSS, no JavaScript yet. Just the structure.
```

### Reading the output

AI returned roughly what I asked for. The parts I changed, with reasons:

- Used `<div>` for the hero. Changed to `<section id="biography">` since hero IS the biography section on my site. Same element, two jobs.
- Generic class names like `.section-1` and `.section-2`. Renamed to `.biography`, `.projects`, `.articles`, `.gallery`. Class names should read like English, not like CSS-ordering.
- Two placeholder project cards. Replaced with one real card (Digital Footprinter) and left room for more. I'd rather show one real project than three lorem-ipsum ones.
- Lorem ipsum intro. Rewrote in my own voice referencing Next Chapter. The whole point of the portfolio is the voice. Lorem ipsum is the enemy.
- A contact form section. Deleted. Out of scope per the scope doc. Footer links cover it.
- Hardcoded social media URLs as `#`. Replaced with my real GitHub and LinkedIn URLs. Site is live, placeholders embarrass.

### Real content swaps

- Name and title: Phil Anthony, Software Engineer.
- Intro paragraph: the one that positions me as a Next Chapter student and frames the bootcamp's premise (AI as daily collaborator, not bolt-on feature).
- Project card: Digital Footprinter, with the real screenshot, real description (browser fingerprinting tool that shows visitors what trackers can read), real stack line, and real links to the live site and GitHub source.
- Articles: the two Substack posts I have live so far ("How I Got Into the Next Chapter AI Builder Bootcamp" and "Rebuilding a Scroll Spy with the Intersection Observer API").
- Footer: real GitHub (`philay3`) and LinkedIn URLs.

### What I learned

Reading AI output is its own skill. The output is almost never wrong, but it's almost never specific to me either. Generic class names, placeholder content, default sections that don't fit my actual scope. Treating it as a first draft instead of a final answer made it useful.

The fastest way to ruin AI-scaffolded code is to ship it with the placeholders intact. The fastest way to make it your own is to do the lorem ipsum kill in the first edit pass, before any styling, before any JavaScript. Voice first, structure second.
