### Notes

Exercise 4 was styling the HTML scaffold from yesterday. The lesson the bootcamp was pushing here was that AI defaults to the fanciest tool it knows, and your job is to keep the structure and trim the flex. Before I asked AI for any CSS, I locked in the visual direction myself so the design stayed mine.

### Visual direction

The palette I settled on is called Balsam & Cedar:

- Background: warm cream (`#f5f0e6`). I wanted the projects and portrait to feel grounded on something paper-like, not floating on white or buried in dark mode.
- Body text: deep green-black (`#1f2d22`). High contrast on the cream without going to pure black.
- Link and emphasis: brick red (`#a8312a`). One warm accent for links and the active nav state.
- Secondary accent: cedar green (`#2d4a37`). Used for button hover and article card hover, not for body content.
- Border and divider: a muted tan (`#d4c9b3`) for header bottom, project image frames, and footer top.

Beyond color:

- Typography: Fraunces for headings (a tighter, characterful serif), Inter for body and UI. Both from Google Fonts with system fallbacks.
- Layout: centered with a max-width of 1100px. Comfortable to read on a laptop without going so wide that the projects section sprawls.
- Sections: generous vertical padding (5rem on biography, 3rem on the others) so each section feels like its own beat.
- Nav: sticky to the top of the viewport with a `z-index` of 100 so it stays reachable from anywhere on the page.

### Prompt I used

```
Write the CSS for my portfolio. Design constraints:

Colors (palette: Balsam & Cedar)
- Background: #f5f0e6
- Text: #1f2d22
- Muted text: #5c5247
- Accent (links, active nav): #a8312a
- Secondary accent (hover states): #2d4a37
- Border: #d4c9b3

Typography
- Headings: 'Fraunces' (Google Fonts), serif fallback
- Body: 'Inter' (Google Fonts), system sans-serif fallback

Layout
- Max content width: 1100px, centered
- Biography section: 5rem top/bottom padding
- Other sections (Projects, Articles, Gallery): 3rem top/bottom padding
- Nav: sticky at the top, horizontal layout

Project cards
- 2-column grid: image on the left, content on the right
- Stack to a single column below 700px wide

Class names already in the HTML:
.biography, .projects, .articles, .gallery, .project, .site-nav, .site-footer

Keep it readable. No fancy animations yet.
```

### What I accepted, modified, rejected

- Sticky nav with `position: sticky` and `z-index`. Accepted. One CSS property each, no JavaScript needed.
- Flexbox for nav alignment. Accepted. Already comfortable with flex from earlier weeks.
- Project cards as a 2-column grid using `grid-template-columns: 1fr 1fr`. Accepted, because it gives equal weight to the screenshot and the description.
- Hover states with `transition: all 0.3s`. Modified. Replaced `all` with specific properties (`background-color`, `color`, `border-color`) so unrelated property changes don't animate. Faster too.
- `clamp()` for responsive font sizing on every heading. Modified. Kept it on the H1 and the article card thumb only; switched everything else to plain `rem` values I could reason about.
- Auto-generated media queries for every section. Modified. Trimmed to one breakpoint (700px, for stacking the project grid). Most of the layout is already fluid because it uses flexbox and `max-width`.
- `cursor: pointer` on the body. Rejected. Made the whole page feel clickable. Deleted.

### What I learned

The biggest lesson here was the same one from Week 2's refactoring exercise. AI defaults to the fanciest tool it knows: `clamp()` on everything, `transition: all`, eight media queries. My job was to keep the structure and trim the flex.

Locking in the visual direction before asking for code mattered more than the prompt itself. When I had specific hex codes, font names, and pixel values, the output was usable on the first pass. When I'd been vague in earlier attempts (asking for a "dark mode aesthetic" or a "minimal portfolio look"), the output was generic and I'd burn time rewriting it.

The constraint isn't the limit, it's the lever.
