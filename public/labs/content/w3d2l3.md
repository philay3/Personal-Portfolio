### Notes

Checkpoint 2 was the lunch-time milestone: the site looks intentional now. Colors, type, spacing, and section rhythm in place. Plus a responsive pass at the same time so mobile isn't an afterthought.

### What changed during the CSS pass

- Typography: loaded Inter (body) and Fraunces (headings) from Google Fonts, with system fallbacks. Set a comfortable 1.6 line-height on body copy.
- Spacing: generous vertical padding on every section so each one reads as its own beat. Centered content with a 1100px max-width so reading length stays comfortable.
- Nav: sticky to the top of the viewport, horizontal on desktop, with the accent color on the active state.
- Project card: 2-column grid (image left, content right), subtle border, rounded corners. Stack to a single column on mobile.
- Footer: simplified to two social icons and a copyright line. Resisted the temptation to overdesign it.

### Responsive pass

Tested in Chrome DevTools device toolbar at 375px (iPhone-ish) and 768px (tablet-ish) widths. The nav wraps, the column reflows, the project card stacks to single-column. One breakpoint handled everything I needed:

```css
@media (max-width: 700px) {
    .project {
        grid-template-columns: 1fr;
    }
}
```

Resisted adding more. Most of the layout is already fluid because it uses flexbox and `max-width` instead of fixed dimensions. The fixed-width pieces (the avatar at 180px, the project card image grid) were the only things the one breakpoint needed to address.

### What I learned

AI defaults to the fanciest tool it knows. The first draft of the CSS came back with eight media queries, `clamp()` everywhere, and `transition: all 0.3s` on every hover state. My job was to keep the structure and trim the flex. Specific over general. One breakpoint over five.

`transition: all` is a particularly hidden trap. It animates unrelated property changes when the browser later updates other properties, which produces janky-looking pages without any obvious cause. Specifying which properties to transition is one extra word and saves debugging time later.
