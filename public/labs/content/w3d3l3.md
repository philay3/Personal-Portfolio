### The case-sensitivity check

One thing I always double-check after a deploy: file paths and capitalization. Local file systems on macOS are case-insensitive by default. Vercel's build environment is case-sensitive. If I'd written `images/PortyPic.png` in the HTML but named the file `portypic.png` on disk, it would have worked locally and 404'd in production.

Verified by inspecting the Network tab on the live site for any failed image or asset requests. Clean.

### What I learned

The workflow is: commit, push, site updates in ~30 seconds. No FTP, no manual deploy step, no build pipeline to maintain. Once it's set up it disappears, which is the mark of good tooling.

The custom domain matters more than I expected. The site has its own identity instead of living on a `*.vercel.app` subdomain. The URL on my resume is one I picked, not one assigned to me. Small but real.
